You are a senior Android engineer. Generate a clean, production-ready Kotlin Android Studio project skeleton for a Cooking Recipes mobile app.

The app must follow modern Android architecture and compile-ready best practices.

=========================================================
CORE REQUIREMENTS
=========================================================

This is a mobile app with:

- Email + password authentication (Firebase Auth)
- Offline-first design
- Local Room database as the source of truth
- Retrofit for external API calls (free recipe API)
- MVVM architecture
- Jetpack Compose UI
- Repository pattern
- Domain/use-case layer
- DataStore (encrypted) for session/token persistence
- No PostgreSQL
- No custom backend required

=========================================================
APP FEATURES
=========================================================

Users can:

1) Maintain a pantry
   - Manual ingredient entry only
   - Exact quantity (Decimal)
   - Single unit per ingredient
   - No expiration tracking

2) Search recipes
   - Show recipes that match pantry
   - Include “almost there” recipes
   - Sort by minimal missing ingredients
   - No dietary filters required

3) View recipe details
   - Ingredients
   - Steps
   - Image (optional)

4) Cook a recipe
   - User scales servings
   - Default amounts scaled automatically
   - User edits actual used amounts
   - Can mark ingredient as MISSING
   - Can SUBSTITUTE with pantry item
   - Pantry decremented ONLY after “Finish Cooking”
   - Must generate audit log entry
   - All changes must occur in ONE Room transaction

5) View cook history
   - CookSession list
   - View usage + audit adjustments

6) Create/edit recipes
7) Import recipe from URL
8) Optionally fetch recipes from free public API and cache locally

=========================================================
AUTHENTICATION
=========================================================

Use Firebase Authentication (Email + Password).

Requirements:
- Login screen
- Sign up screen
- Logout
- Gate entire app behind authentication
- Store minimal user info locally (userId only)
- Store session using DataStore (encrypted if possible)
- Use Firebase SDK properly
- No custom auth server

=========================================================
